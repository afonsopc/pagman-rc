O Modelo OSI

O modelo OSI (Open System Interconnection) é um modelo conceitual criado pela Organização Internacional de Normalização que permite que
diversos sistemas de comunicação se comuniquem usando protocolos
padronizados. Em poucas palavras, o modelo OSI fornece um padrão para que diferentes sistemas de computadores comunicar entre si.
O modelo OSI pode ser visto como uma linguagem universal para as Redes de computadores.
É baseia-se no conceito de dividir um sistema de comunicação em sete camadas abstratas, empilhadas umas em cima das outras.

As Camadas

  • Camada Física

    • Esta camada define as características do meio físico de transmissão da rede, conectores, interfaces ou modulação de sinais.

    • RS-232, RJ45, V.34, 100BASE-TX, SDH, DSL, 802.11

  • Camada de Dados

    • Esta camada controla o acesso ao meio físico de transmissão. (Controlo de erros da camada física)

    • Ethernet, 802.11, MAC/LLC, VALN, ATM, HDP, Fibra, Frame Relay, HDLC, PPP, Q.921, Token Ring

  • Camada de Rede

    • Esta camada usa o endereçamento lógico (Endereços de IP) para enviar e receber dados. (Encaminhamento de pacotes)

    • IP, ARP, IPsec, ICMP, IGMP, OSPF

  • Camada de Transporte

    • Esta camada é responsável por encaminhar os dados pelas redes, controla o fluxo de informação, segmentação e o controlo de erros.

    • TCP, UDP, SCTP, SSL, TLS

  • Camada de Sessão

    • Esta camada controla (estabelece, faz gestão e termina) as sessões entre aplicações.

    • Sessão estabelecida em TCP, SIP, RTP, RPC-Named pipes

  • Camada de Apresentação

    • Esta camada assegura a compatibilidade entre as camadas de aplicação de sistemas diferentes.

    • HTML, DOC, JPEG, MP3, AVI, Sockets

  • Camada de Aplicação

    • Esta camada fornece serviçoes às aplicações do utilizador.

    • DNS, WWW/HTTP, P2P, EMAIL/POP, SMTP, Telnet, FTP

Exemplo prático

Vamos ver como é que esta página chegou ao seu computador.

  1. A camada de aplicação (Browser), serve como uma interface para apresentar a página.

  2. A camada de apresentação é onde o ficheiro HTML desta página se situa, que será recebido no pedido a seguir.

  3. Imaginaremos que o pedido do browser para o servidor foi feito a usar o protocolo HTTP.

  4. Na camada de sessão, é onde a sessão é estabelecida entre o seu computador (Browser) e o servidor (Website).

  5. Na camada de transporte, o protocolo TCP fornece garantia na entrega de todos os pacotes enviados entre o seu computador e o servidor.

  6. Tanto o seu computador e o servidor têm um endereço de IP, isto é uma funcionalidade da camada de rede.

  7. A camada de dados, é onde o seu endereço de IP será traduzido para o endereço MAC (endereço da placa de rede).

  8. Agora na camada física são os cabos de cobre, fibra óptica, placas de rede, hubs, e outros dispositivos, ajudam nesta ligação física entre o seu computador e o servidor.

Equipamentos tradicionais e suas camadas

  • Switch funciona na camada de dados. (Também poderá funcionar em vez na camada de rede)

  • Hub funciona na camada física.

  • Router funciona na camada de rede.

  • Browser funciona na camada de aplicação.

  • Ficheiros HTML funcionam na camada de apresentação.

  • Cabos fibra óptica funcionam na camada física.

  • Ethernet funciona na camada de dados.

  • O cliente de email funciona na camada de aplicação.

  • Ficheiros de Vídeo funcionam na camada de apresentação.
